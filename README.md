# 한희경 AI 강사 공식 홈페이지

명함 뒷면 QR코드와 연결할 가로형 반응형 프로필 홈페이지입니다.
PC에서는 왼쪽 사진 + 오른쪽 정보(가로형), 모바일에서는 위아래로 자연스럽게 바뀝니다.

---

## 1. 파일 설명

| 파일 | 설명 |
|---|---|
| `index.html` | 홈페이지 본문 구조 |
| `style.css` | 색상·레이아웃·반응형 디자인 |
| `script.js` | SNS 버튼, AI 도구 목록, 링크 복사 기능 |
| `profile.png` | 강사 프로필 사진 |
| `qr-code.png` | 홈페이지 주소로 만든 QR코드 (왼쪽 사진 패널에 삽입됨) |
| `README.md` | 이 설명서 |

---

## 2. profile.png 사진 교체 방법

1. 실제 프로필 사진을 준비합니다 (세로형 사진 권장, 예: 900×1200px).
2. 파일 이름을 반드시 `profile.png`로 저장합니다.
3. 기존 `profile.png`를 새 사진으로 덮어씁니다.
4. 사진을 찾지 못해도 페이지가 깨지지 않고 이니셜(한)로 자동 표시됩니다.

---

## 3. SNS 주소 변경 방법

`script.js` 파일 맨 위 `SNS_LINKS` 배열 하나만 고치면 전체 페이지에 반영됩니다.

```js
const SNS_LINKS = [
  { name: "유튜브", desc: "...", url: "여기에 새 주소", icon: "..." },
  ...
];
```

전화번호·이메일은 `index.html`에서 `tel:`, `mailto:` 부분을 찾아 수정합니다.

---

## 4. GitHub 저장소에 파일 올리는 방법

1. [github.com](https://github.com) 로그인 → 오른쪽 위 **+** → **New repository**
2. 저장소 이름 예: `han-ai-profile` (자유롭게 정해도 됩니다)
3. **Public** 선택 → **Create repository**
4. 저장소 페이지에서 **Add file → Upload files**
5. `index.html`, `style.css`, `script.js`, `profile.png`, `qr-code.png`, `README.md` 6개 파일을 통째로 끌어다 놓기
6. 아래로 내려가서 **Commit changes** 클릭
7. ⚠️ 파일들이 폴더 안이 아니라 저장소 **최상위**에 바로 있어야 합니다

---

## 5. GitHub Pages 설정 방법

1. 저장소 상단 **Settings** 클릭
2. 왼쪽 메뉴에서 **Pages** 클릭
3. **Source**에서 **Deploy from a branch** 선택
4. **Branch**는 `main` 선택, **Folder**는 `/(root)` 선택
5. **Save** 클릭

---

## 6. 홈페이지 주소 확인 방법

1~3분 기다렸다가 Pages 화면을 새로고침하면 초록색으로 아래와 비슷한 주소가 나타납니다.

```
https://깃허브아이디.github.io/저장소이름/
```

---

## 7. 홈페이지 주소로 QR코드 만드는 방법

⚠️ **가짜 QR코드는 넣지 않았습니다.** 홈페이지 왼쪽 사진 패널에 있는 QR코드는
실제 배포 주소(`https://queenofqn7.github.io/han-ai-profile/`)로 이미 생성된 것입니다.
만약 저장소 이름이나 GitHub 아이디가 달라져서 실제 주소가 바뀌면, 아래 방법으로 QR코드를 새로 만들어 `qr-code.png`를 교체해야 합니다.

1. QR코드 생성 사이트(예: qr-code-generator.com, me-qr.com 등)에 접속
2. 6번에서 확인한 실제 주소를 입력
3. 고해상도 PNG로 다운로드 (인쇄용은 300dpi 이상 권장)
4. 필요하면 투명 배경 옵션을 선택
5. 인쇄용으로는 SVG(벡터) 다운로드가 있으면 SVG로도 받아두면 좋습니다 (확대해도 깨지지 않음)

## 8. 명함에 QR코드를 넣을 때 주의사항

- QR코드 주변에는 흰색 여백을 충분히 남겨주세요 (여백이 없으면 인식이 잘 안 됩니다)
- QR코드 아래에 홈페이지 주소를 작은 글씨로 함께 표기해 주세요
- 명함을 인쇄하기 전, 실제 인쇄 크기로 출력해서 스캔 테스트를 한 번 해보세요
- 어두운 배경 위에 QR코드를 올릴 경우 명암 대비가 충분한지 확인해 주세요

---

## 9. 수정 후 다시 배포하는 방법

1. 수정할 파일(`index.html`, `style.css`, `script.js`, `profile.png` 등)을 로컬에서 수정합니다.
2. GitHub 저장소 페이지에서 수정한 파일을 다시 업로드하거나,
   해당 파일을 열어 연필(✏️) 아이콘 → 내용 수정 → **Commit changes**를 클릭합니다.
3. GitHub Pages는 커밋 후 자동으로 다시 배포됩니다 (보통 1~3분 소요).
4. 홈페이지 주소를 강력 새로고침(Ctrl+Shift+R / Cmd+Shift+R)해서 반영을 확인합니다.
5. 이미 인쇄된 QR코드는 그대로 사용 가능합니다 (주소가 바뀌지 않는 한 다시 만들 필요 없음).
